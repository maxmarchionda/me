# Project Brief

## Design Intent

A polished personal portfolio for an applied-AI engineering leader: technically credible,
direct, and composed. Dark mode is product-forward; light mode is warmer and more personal.
Animation clarifies hierarchy and system behavior without turning the site into a component demo.

## Audience

Engineering and business leaders evaluating senior technical leadership, collaborators with hard
product problems, and peers. Visitors need fast evidence of scope, execution, and business fluency.

## Voice and Tone

Direct, grounded, technically fluent, and warm. Prefer specific outcomes over generic claims.

## Constraints

- Preserve the existing light/dark semantic token seam and no-flash theme initialization.
- One icon family (Lucide) and one deterministic tween motion language.
- SmoothUI and Magic UI code is adapted locally; imported defaults never bypass project tokens.
- `prefers-reduced-motion` must deliver every state without decorative animation.
